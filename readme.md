# Drone Object Detection: Frameworks & Modular Pipeline

This documentation outlines the utilization, file structure, and training procedures for the modularized object detection pipeline. The monolithic Jupyter Notebook has been broken down into 6 distinct, maintainable Python scripts. These scripts handle the BIRDSAI dataset to train, evaluate, and visually compare **YOLOv8**, **Faster R-CNN**, and **Deformable DETR**.

---

## 📁 Required File Structure

To run these scripts successfully (e.g., in a Kaggle environment or locally), your working directory must follow this structure. The scripts assume they are placed in the same root directory.


```
/kaggle/working/ (or your local root)
│
├── data_preparation.py      # Script 1: Converts MOT to YOLO format
├── dataset_utils.py         # Script 2: PyTorch Dataloader utilities
├── yolo_pipeline.py         # Script 3: YOLOv8 Training/Eval
├── frcnn_pipeline.py        # Script 4: Faster R-CNN Training/Eval
├── detr_pipeline.py         # Script 5: DETR Training/Eval
├── master_comparison.py     # Script 6: Final visual grid generation
│
└── yolo_birdsai/            # AUTO-GENERATED by data_preparation.py
    ├── dataset.yaml
    ├── images/
    │   ├── train/
    │   ├── val/
    │   └── test/
    └── labels/
        ├── train/
        ├── val/
        └── test/
```

## 🚀 Execution Guide & Training Procedure

Follow these steps in order to train and evaluate the models.

### Step 1: Data Preparation
**File:** `data_preparation.py`
**Purpose:** Reads the raw MOT-formatted `.csv` annotations, maps them to the corresponding images, normalizes the bounding boxes to `[x_center, y_center, width, height]`, and splits them into train/val/test directories. It also generates the `dataset.yaml` required by YOLO.

**How to run:**
```python data_preparation.py```
*Wait for this to finish before running any other scripts, as the models rely on the `yolo_birdsai` directory.*

### Step 2: YOLOv8 Pipeline
**File:** `yolo_pipeline.py`
**Purpose:** Utilizes the Ultralytics library to train a baseline YOLOv8 model and evaluates it using `torchmetrics` to generate Small/Medium/Large mAP metrics.

**How to run:**
```python yolo_pipeline.py```
*Note: To toggle between training and evaluating, uncomment/comment `train_baseline()` and `test_evaluation_yolo()` in the `if __name__ == '__main__':` block.*

### Step 3: Faster R-CNN Pipeline
**File:** `frcnn_pipeline.py`
**Purpose:** Builds a Custom ResNet-50 backbone with an explicit Feature Pyramid Network (FPN). It includes a monkey-patch to replace standard Cross Entropy loss with Focal Loss.
**Dependencies:** Imports `BIRDSAIYOLODataset` from `dataset_utils.py`.

**How to run:**
```python frcnn_pipeline.py```
*Note: You will need to define your training loop triggers at the bottom of the file (e.g., calling `run_training_experiment(experiment_type="FL")`).*

### Step 4: Deformable DETR Pipeline
**File:** `detr_pipeline.py`
**Purpose:** Fine-tunes a Deformable DETR model from Hugging Face. It contains logic to freeze the backbone and selectively freeze the Encoder (Exp2) or Decoder (Exp3) to test different transfer learning strategies.
**Dependencies:** Imports `BIRDSAIYOLODataset` from `dataset_utils.py`.

**How to run:**
```python detr_pipeline.py```
*Note: You will need to trigger `run_all_detr_experiments()` at the bottom of the script.*

### Step 5: Visual Master Comparison
**File:** `master_comparison.py`
**Purpose:** The culmination of the project. It loads the best saved weights from YOLO, Faster R-CNN (CE and FL), and DETR (Pretrained, Exp1, Exp2, Exp3), runs them on targeted "Easy" and "Hard" images, and plots a 2x4 master comparison grid.
**Dependencies:** Requires all models to have saved weights on disk.

**How to run:**
```python master_comparison.py```

## 🔧 Utility Breakdown

### `dataset_utils.py`
You **do not** run this script directly. 
It acts as a bridging module. Because we converted the raw data to YOLO `.txt` format in Step 1, Faster R-CNN and DETR natively don't know how to read it. This script contains the `BIRDSAIYOLODataset` PyTorch class that reads images and YOLO `.txt` labels on-the-fly and converts them into the absolute `[x_min, y_min, x_max, y_max]` tensor dictionary format expected by `torchvision` models.

## 📝 Troubleshooting & Notes
1. **CUDA Out of Memory:** If you encounter RAM or VRAM crashes during Faster R-CNN or DETR training, reduce the `batch_size` in the `DataLoader` setups from `4` to `2`.
2. **Paths:** Ensure the file paths in the `WEIGHTS` dictionaries inside `master_comparison.py` point exactly to where your models saved their `.pt` or `.pth` files.
